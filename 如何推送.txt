📌 由于Cursor IDE终端环境限制，请使用以下方式推送到GitHub
================================================================

Cursor IDE的内置终端无法识别Git命令（即使系统已安装）
这是正常的环境隔离问题，需要在外部执行。

================================================================
✅ 推荐方法：双击批处理文件（最简单）
================================================================

在文件资源管理器中找到并双击：

    📄 立即推送.bat

这个文件会自动完成所有Git操作！


================================================================
🔄 备选方法：使用外部PowerShell
================================================================

1. 按 Win + X
2. 选择 "Windows PowerShell" 或 "终端"
3. 复制粘贴以下所有命令（一次性执行）：

cd D:\cursor\expense-dashboard
git init
git add .
git commit -m "Initial commit: 费用Dashboard项目"
git remote add origin https://github.com/irenechen261/expense.git
git branch -M main
git push -u origin main --force


================================================================
⚠️ 身份验证说明
================================================================

推送时可能需要GitHub身份验证：

方法1：使用GitHub桌面客户端（推荐）
   - 会自动弹出登录窗口
   - 登录后即可推送

方法2：使用Personal Access Token
   1. 访问：https://github.com/settings/tokens
   2. 点击 "Generate new token (classic)"
   3. 勾选 "repo" 权限
   4. 生成并复制token
   5. 推送时用token作为密码


================================================================
✅ 如何确认推送成功
================================================================

访问：https://github.com/irenechen261/expense

如果看到项目文件（app/, components/, lib/ 等），说明推送成功！


================================================================
📊 推送成功后的下一步：部署到Vercel
================================================================

1. 访问 https://vercel.com
2. 点击 "Sign Up" 或 "Login"
3. 选择 "Continue with GitHub" 登录
4. 点击 "Add New..." → "Project"
5. 找到并选择 "irenechen261/expense"
6. 点击 "Import"
7. 保持所有默认设置，直接点击 "Deploy"
8. 等待2-3分钟
9. 获得生产URL：https://expense-xxxx.vercel.app

详细说明请查看：Vercel部署指南.md


================================================================
💡 为什么不能在Cursor中直接执行？
================================================================

Cursor IDE使用沙盒环境来运行终端命令，这个环境：
- 有独立的PATH环境变量
- 不会自动继承系统的Git路径
- 需要重启IDE才能加载新安装的工具

解决方案：
1. 使用外部工具（推荐）
2. 或完全关闭并重启Cursor IDE（然后Git可能会被识别）
